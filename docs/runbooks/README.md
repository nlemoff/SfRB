# Runbooks

Operational runbooks for diagnosing and recovering from common situations.

- [Bridge troubleshooting](./bridge-troubleshooting.md) — the bridge won't start, won't save, or won't reach the AI provider.
- [Incident response](./incident-response.md) — how to triage and respond when something is broken in `main` or a release.
- [Release](./release.md) — how releases are cut and published.

Each runbook lists symptoms, likely causes, and concrete remediation steps. Keep them short and action-oriented; link to deeper docs rather than duplicating them.
