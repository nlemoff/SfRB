# S01: CLI & Config Agent — UAT

**Milestone:** M001
**Written:** 2026-03-13

## UAT Type

- UAT mode: artifact-driven
- Why this mode is sufficient: S01 ships a CLI/config workflow with explicit filesystem outputs and diagnostics, so correctness is proven by command behavior, generated files, and redaction/failure signals rather than subjective UX judgment.

## Preconditions

- Run from the SfRB repo root.
- Dependencies are installed (`npm install` already completed for the workspace).
- `dist/cli.js` exists or can be generated by the smoke script/build.
- Use a throwaway temp workspace for tests that write `sfrb.config.json`.
- Do not paste a real provider secret; use test strings such as `sk-test-123` or `sk-ant-test-123`.

## Smoke Test

Run:

`node scripts/verify-s01-init-smoke.mjs`

Expected:
- script exits 0
- output contains `S01 init smoke check passed.`
- output prints a temp workspace path and config path
- no raw API key appears in stdout/stderr

## Test Cases

### 1. Fresh workspace interactive-equivalent init succeeds

1. Create a fresh temp directory.
2. Run `node ./dist/cli.js init --cwd <temp-dir>` with `SFRB_INIT_TEST_INPUT` set to JSON like `{"provider":"anthropic","apiKey":"sk-ant-test-123","physics":"design","confirm":true}`.
3. Inspect `<temp-dir>/sfrb.config.json` and `<temp-dir>/.gitignore`.
4. **Expected:** command exits 0, `sfrb.config.json` exists, config contains `ai.provider = "anthropic"`, `ai.apiKeyEnvVar = "ANTHROPIC_API_KEY"`, `workspace.physics = "design"`, `.gitignore` contains `sfrb.config.json`, and command output does not contain `sk-ant-test-123`.

### 2. Non-interactive flag-driven init succeeds

1. Create a second fresh temp directory.
2. Run `node ./dist/cli.js init --cwd <temp-dir> --provider openai --api-key sk-openai-test-123 --physics document --yes`.
3. Read the generated config file.
4. **Expected:** command exits 0, config contains `ai.provider = "openai"`, `ai.apiKeyEnvVar = "OPENAI_API_KEY"`, `workspace.physics = "document"`, `.gitignore` contains `sfrb.config.json`, and the raw key is absent from command output.

### 3. Invalid provider and physics are rejected with field-level diagnostics

1. Run `node ./dist/cli.js init --provider invalid --physics nope`.
2. Capture stderr and the exit code.
3. **Expected:** command exits non-zero and stderr names the rejected fields `ai.provider` and `workspace.physics` in structured validation output.

### 4. Cancellation does not write files

1. Create a fresh temp directory.
2. Run `node ./dist/cli.js init --cwd <temp-dir>` with `SFRB_INIT_TEST_INPUT` set to `{"provider":"openai","apiKey":"sk-test-123","physics":"document","confirm":false}`.
3. Inspect the temp directory contents.
4. **Expected:** command exits non-zero with a clear cancellation message, and neither `sfrb.config.json` nor `.gitignore` is created or modified by the canceled run.

### 5. Non-TTY interactive misuse fails explicitly

1. Create a fresh temp directory.
2. Run `node ./dist/cli.js init --cwd <temp-dir>` in a non-interactive context without `SFRB_INIT_TEST_INPUT` and without required flags.
3. Capture stderr.
4. **Expected:** command exits non-zero with an actionable message indicating interactive input is unavailable or required parameters are missing, and no config file is written.

## Edge Cases

### Existing `.gitignore` gains protection without losing prior content

1. In a fresh temp directory, create a `.gitignore` containing unrelated entries such as `node_modules/`.
2. Run a successful init flow.
3. **Expected:** `.gitignore` still contains the original entries and now also contains `sfrb.config.json`.

### Secret redaction in summary output

1. Run a successful init flow using a recognizable fake key like `sk-visible-check-999`.
2. Review stdout/stderr and any test logs.
3. **Expected:** the exact string `sk-visible-check-999` never appears; only redacted or metadata-only output is shown.

## Failure Signals

- `sfrb.config.json` is missing after a reported successful init.
- `.gitignore` does not contain `sfrb.config.json` after init.
- stdout/stderr contains the raw API key used in the test.
- invalid provider/physics values do not produce field-level validation messages.
- cancellation or non-TTY failure still leaves behind config artifacts.

## Requirements Proved By This UAT

- R002 — the workspace can be initialized with BYOK provider metadata and provider-specific key handling through `sfrb init`.
- R003 — the workspace physics preference is captured and persisted in a stable local config contract.

## Not Proven By This UAT

- R004 — no local server or browser bridge is exercised here.
- R005 — no canvas editing, document mutation, or layout behavior is exercised here.
- Real human prompt feel is only lightly covered because automated harness input is the primary verification path for this slice.

## Notes for Tester

- Prefer fake secrets in all manual checks.
- If `dist/cli.js` is missing, run `npm run build` first.
- The automation harness path (`SFRB_INIT_TEST_INPUT`) is intentional and is part of the shipped slice, not a test-only mock of internal prompt behavior.
